Wake Development Roadmap

1. Type Tags

Type tags might prove to be a flagship feature of Wake, but to do them right and/or get their whole benefit involves several moving parts.

Parameterized Ranges

Declaring generic classes with restrictions on their parameterized types, ie, Cone{T from Icecream}.

Incredibly useful for generic programming, and should be available for std lib development/design.

Self Type

A special generic a type that represents this. Extending the class, self becomes the extension.

Type Tags

A freeform typechecked label for distinguishing types with the compiler's help. Example: keeping Num:miles and Num:kilometers apart.

Typesafe Autoboxing

The standard library will now be able to have methods such as Self -- maxOf(Self)

Service Extensions

Classes that add methods to other classes when in scope. This is nice for extending the std library, or other libraries, and can lead to cleaner tests to boot.

Operator overloading

Instead of adding methods to other classes, they create operator overloading. Combined with type tags, this can be used to automatically convert units on arithmatic/assignment.
